# euissGeo Package Structure

## Recommended Directory Structure:
```
euissGeo/
├── DESCRIPTION
├── NAMESPACE  
├── LICENSE
├── README.md
├── R/
│   ├── data_access.R          # GISCO and data retrieval
│   ├── coordinate_utils.R     # Coordinate conversion functions
│   ├── spatial_operations.R  # Spatial analysis functions
│   ├── cartographic.R         # Mapping and visualization
│   ├── raster_utils.R        # Raster processing
│   └── general_utils.R       # General utilities
└── man/ (generated by roxygen2)
```

## Function Organization:

### data_access.R
- `euiss_gisco()` - Main GISCO data retrieval
- `euiss_gisco_kosovo()` - Kosovo-specific handling
- `euiss_load_raster()` - Raster data loading

### coordinate_utils.R  
- `euiss_coords_to_sf()` - Convert coordinates to SF
- `euiss_coords_char2dec()` - Parse character coordinates
- `euiss_sf_to_coords()` - Extract coordinates from SF
- `euiss_geocode()` - Geocoding with OSM

### spatial_operations.R
- `euiss_left_join()` - Spatial joins with country codes
- `euiss_crop_raster()` - Crop rasters to geometries

### cartographic.R
- `euiss_basemap()` - Complex base map creation
- `euiss_cartogram()` - Cartogram generation  
- `euiss_spikemap()` - Spike map visualization
- `geom_relief_euiss()` - Elevation visualization (could stay in euissR)

### general_utils.R
- `euiss_substring()` - String manipulation
- `euiss_unzip()` - Interactive file extraction
- `euiss_date_from_excel()` - Excel date conversion
- `euiss_locator()` - Interactive point selection

## Integration with euissR:

All mapping functions should use euissR colors and themes:

```r
# Example: Updated euiss_basemap() function
euiss_basemap <- function(..., 
                          colors = list(
                            water = euissR::get("teal", envir = euissR::.euiss_env),
                            borders = euissR::get("col_axis", envir = euissR::.euiss_env),
                            urban = euissR::get("col_grid", envir = euissR::.euiss_env)
                          )) {
  # Function implementation using euissR styling
}
```

## Benefits of This Structure:

1. **Clear separation of concerns**
2. **Manageable file sizes** 
3. **Easy to find functions**
4. **Consistent naming patterns**
5. **euissR integration for branding consistency**
